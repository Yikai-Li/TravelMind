Agent Pipeline Details

ConstraintParser Agent

Input: Raw user data

  • Parse dates ‚Üí date range
  • Normalize budget ‚Üí daily budget
  • Parse interests ‚Üí list
  • Detect conflicts ‚Üí warnings
  • Fill gaps ‚Üí assumptions

Output: Structured constraints object

‚Üì

DestinationRecommender Agent

Input: Parsed constraints

  • Build LLM prompt with constraints
  • Query GPT-4 for recommendations
  • Parse JSON response
  • Score destinations (0-100)
  • Add distance/cost estimates

Output: Ranked destination list

‚Üì

ItineraryPlanner Agent

Input: Top destination + constraints

  • Calculate trip duration
  • Build day-by-day structure
  • Allocate activities by pace
  • Add meal suggestions
  • Include transportation

Output: Day-by-day itinerary skeleton

‚Üì

DetailEnricher Agent

Input: Itinerary skeleton

  • For each activity:
    • Add specific timing
    • Estimate costs
    • Add booking info
    • Include tips
  • Add hotel recommendations (3-5)
  • Add restaurant options
  • Include transport details

Output: Fully detailed itinerary

‚Üì

DualModelEnricher Agent

Input: Activity details

  • Use travel-agent model for insights
  • Use GPT for structured data
  • Web scraping for validation
  • Combine multiple sources

Output: Enriched activity details

‚Üì

PlanEnhancer Agent (Dual-Model)

Input: Existing plan text

  • Thread 1: HuggingFace model (20s timeout)
  • Thread 2: GPT-4 enhancement
  • Combine insights from both
  • Format as structured JSON
  • Add hotels, restaurants, costs, tips

Output: Enhanced plan with metadata
